# User Stories (MVP)

> 형식: **As a [사용자], I want [기능], so that [가치/이유]**
> 우선순위: P0 (필수) / P1 (중요) / P2 (있으면 좋음)

---

## 1. 랜딩 페이지

| ID | 우선순위 | 스토리 | 인수 조건 |
|----|----------|--------|-----------|
| US-101 | P0 | 방문자로서, 첫 화면에서 브랜드의 대표 제품을 풀스크린 이미지로 보고 싶다. 브랜드의 감성과 품질을 직관적으로 느끼기 위해. | - 히어로 섹션이 뷰포트 전체를 채운다<br>- 브랜드 카피와 CTA 버튼("컬렉션 보기")이 표시된다<br>- CTA 클릭 시 `/products`로 이동한다 |
| US-102 | P1 | 방문자로서, 랜딩 페이지에서 베스트셀러 제품을 바로 확인하고 싶다. 인기 있는 제품을 빠르게 탐색하기 위해. | - 베스트셀러 3~4개가 카드 형태로 노출된다<br>- 카드 클릭 시 해당 제품 상세 페이지로 이동한다 |
| US-103 | P1 | 방문자로서, 브랜드의 철학과 정체성을 간단히 알고 싶다. 구매 전 브랜드를 신뢰하기 위해. | - 브랜드 소개 섹션에 철학 문구와 설명이 표시된다 |

## 2. 네비게이션 & 레이아웃

| ID | 우선순위 | 스토리 | 인수 조건 |
|----|----------|--------|-----------|
| US-201 | P0 | 사용자로서, 어떤 페이지에서든 헤더를 통해 주요 페이지로 이동하고 싶다. 원하는 곳에 빠르게 접근하기 위해. | - 헤더에 로고, 컬렉션, 브랜드 소개 메뉴가 있다<br>- 로고 클릭 시 `/`로 이동한다 |
| US-202 | P0 | 사용자로서, 헤더에서 장바구니 아이콘과 담긴 수량을 확인하고 싶다. 현재 장바구니 상태를 항상 파악하기 위해. | - 장바구니 아이콘에 아이템 수 뱃지가 표시된다<br>- 아이콘 클릭 시 `/cart`로 이동한다<br>- 장바구니가 비어있으면 뱃지가 표시되지 않는다 |
| US-203 | P0 | 모바일 사용자로서, 햄버거 메뉴를 통해 네비게이션에 접근하고 싶다. 작은 화면에서도 편하게 이동하기 위해. | - 768px 미만에서 햄버거 아이콘이 표시된다<br>- 클릭 시 슬라이드 드로어가 열린다<br>- 메뉴 항목 클릭 시 드로어가 닫히고 해당 페이지로 이동한다 |
| US-204 | P1 | 사용자로서, 푸터에서 고객센터, 배송/반품 정책, 약관 정보를 확인하고 싶다. 필요한 정보를 쉽게 찾기 위해. | - 푸터에 링크 그룹과 SNS 아이콘이 표시된다 |
| US-205 | P0 | 사용자로서, 헤더에서 로그인 상태를 확인하고 마이페이지로 이동하고 싶다. 내 계정에 빠르게 접근하기 위해. | - 비로그인 시 사용자 아이콘 클릭 → `/login`으로 이동<br>- 로그인 시 사용자 아이콘 클릭 → `/mypage`로 이동 |

| ID | 우선순위 | 스토리 | 인수 조건 |
|----|----------|--------|-----------|
| US-401 | P0 | 고객으로서, 전체 안경 제품 목록을 그리드 형태로 보고 싶다. 한눈에 제품들을 비교하기 위해. | - `/products`에서 제품 카드(이미지, 이름, 가격)가 그리드로 표시된다<br>- 데스크톱 3~4열, 모바일 1~2열 |
| US-402 | P0 | 고객으로서, 카테고리별로 제품을 필터링하고 싶다. 원하는 종류의 안경만 빠르게 찾기 위해. | - 선글라스 / 안경 / 블루라이트 차단 필터가 있다<br>- 필터 클릭 시 해당 카테고리 제품만 표시된다<br>- 전체 보기 옵션이 있다 |
| US-403 | P1 | 고객으로서, 제품을 가격순이나 최신순으로 정렬하고 싶다. 내 기준에 맞게 탐색하기 위해. | - 최신순 / 가격 낮은순 / 가격 높은순 정렬이 가능하다<br>- 정렬 변경 시 목록이 즉시 갱신된다 |
| US-404 | P0 | 고객으로서, 제품 카드를 클릭해서 상세 페이지로 이동하고 싶다. 제품 정보를 자세히 보기 위해. | - 카드 클릭 시 `/products/[id]`로 이동한다 |

## 5. 제품 상세

| ID | 우선순위 | 스토리 | 인수 조건 |
|----|----------|--------|-----------|
| US-501 | P0 | 고객으로서, 제품의 여러 앵글 이미지를 확인하고 싶다. 실물에 가까운 느낌을 파악하기 위해. | - 메인 이미지와 썸네일 목록이 표시된다<br>- 썸네일 클릭 시 메인 이미지가 전환된다 |
| US-502 | P0 | 고객으로서, 제품의 상세 정보(설명, 소재, 사이즈)를 확인하고 싶다. 나에게 맞는 제품인지 판단하기 위해. | - 이름, 가격, 설명, 소재, 사이즈 정보가 모두 표시된다 |
| US-503 | P0 | 고객으로서, 원하는 색상을 선택하고 싶다. 내 취향에 맞는 옵션을 고르기 위해. | - 컬러 스와치가 표시된다<br>- 스와치 클릭 시 선택 상태가 표시되고 대표 이미지가 바뀐다 |
| US-504 | P0 | 고객으로서, 수량을 선택하고 장바구니에 담고 싶다. 구매를 진행하기 위해. | - 수량 증가/감소 버튼이 있다 (최소 1)<br>- "장바구니 담기" 클릭 시 장바구니에 추가된다<br>- 추가 완료 피드백(토스트 또는 뱃지 업데이트)이 표시된다 |
| US-505 | P2 | 고객으로서, 상세 페이지 하단에서 관련 제품을 추천받고 싶다. 추가 탐색을 쉽게 하기 위해. | - 같은 카테고리의 다른 제품 2~3개가 하단에 표시된다 |

## 6. 장바구니

| ID | 우선순위 | 스토리 | 인수 조건 |
|----|----------|--------|-----------|
| US-601 | P0 | 고객으로서, 장바구니에 담은 제품 목록을 확인하고 싶다. 주문 전 내역을 검토하기 위해. | - 각 아이템의 이미지, 이름, 선택 옵션(색상), 수량, 금액이 표시된다 |
| US-602 | P0 | 고객으로서, 장바구니에서 수량을 변경하고 싶다. 주문 수량을 조절하기 위해. | - +/- 버튼으로 수량 변경이 가능하다<br>- 수량 변경 시 소계와 총액이 즉시 갱신된다<br>- Supabase `cart_items` 테이블이 동기화된다 |
| US-603 | P0 | 고객으로서, 장바구니에서 특정 아이템을 삭제하고 싶다. 불필요한 제품을 제거하기 위해. | - 각 아이템에 삭제 버튼이 있다<br>- 삭제 시 목록에서 즉시 제거되고 총액이 갱신된다 |
| US-604 | P0 | 고객으로서, 주문 요약(소계, 배송비, 총액)을 확인하고 싶다. 최종 결제 금액을 알기 위해. | - 소계, 배송비, 총액이 표시된다<br>- 배송비는 고정 금액 또는 무료 배송 기준으로 표시된다 |
| US-605 | P0 | 고객으로서, 장바구니가 비어있을 때 안내를 받고 싶다. 현재 상태를 명확히 인지하기 위해. | - 빈 장바구니일 때 안내 문구와 "쇼핑하러 가기" 버튼이 표시된다 |
| US-606 | P0 | 고객으로서, 다른 기기에서도 장바구니가 유지되길 원한다. 어디서든 이어서 쇼핑하기 위해. | - 로그인 사용자의 장바구니는 Supabase에 저장된다<br>- 다른 기기에서 로그인 시 동일한 장바구니가 표시된다 |

## 7. 체크아웃 (주문)

| ID | 우선순위 | 스토리 | 인수 조건 |
|----|----------|--------|-----------|
| US-701 | P0 | 고객으로서, 배송 정보(이름, 연락처, 주소)를 입력하고 싶다. 제품을 받을 곳을 지정하기 위해. | - 이름, 연락처, 주소, 우편번호 입력 폼이 있다<br>- 프로필에 저장된 정보가 있으면 자동 입력된다<br>- 필수 필드 미입력 시 유효성 검사 에러가 표시된다 |
| US-702 | P1 | 고객으로서, 배송 메모를 남기고 싶다. 배송 시 참고 사항을 전달하기 위해. | - 배송 메모 입력란이 선택 사항으로 있다 |
| US-703 | P0 | 고객으로서, 결제 수단을 선택하고 싶다. 원하는 방법으로 결제하기 위해. | - 신용카드 / 네이버페이 옵션이 표시된다<br>- (MVP에서는 UI만 구현, 실제 결제 불가) |
| US-704 | P0 | 고객으로서, 주문 전 최종 요약을 확인하고 싶다. 실수 없이 주문을 확정하기 위해. | - 주문 아이템, 배송 정보, 결제 금액이 요약 표시된다 |
| US-705 | P0 | 고객으로서, "주문 완료" 버튼을 눌러 주문을 확정하고 싶다. 구매를 완료하기 위해. | - 모든 필수 정보 입력 후 버튼이 활성화된다<br>- 클릭 시 Supabase에 주문이 저장되고 `/order-complete`로 이동한다 |
